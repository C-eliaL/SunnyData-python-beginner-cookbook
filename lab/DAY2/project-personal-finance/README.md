## 📖 專案描述

### 故事背景

延續課綱 @class.md 單元 19–21，學生要打造「簡易個人理財小幫手」。使用者輸入本月收入與三項支出，程式自動計算是否超支，並給予立即回饋。

### 流程概念

```
開始
  ↓
定義 calculate_savings()
  ↓
輸入收入 & 三項支出
  ↓
計算總支出與儲蓄
  ↓
儲蓄結果判斷
  ↓
顯示鼓勵或提醒
  ↓
結束
```

## 🛠️ 開發指南（漸進式）

### 階段 A：完成儲蓄函式 (30 分鐘)
- **目標**：撰寫 `calculate_savings(income, expense_total)` 並回傳差額。
- **步驟**：
  1. 定義函式，參數為收入與總支出。
  2. 回傳收入減總支出結果。
  3. 使用範例數字呼叫函式驗證。
- **檢核點**：
  - [ ] 呼叫 `calculate_savings(40000, 32000)` 回傳 8000
  - [ ] 函式僅負責計算，不輸出文字
- **提示**：從 @class.md 的 `calculate_savings` 範例延伸即可。

### 階段 B：建立輸入流程 (45 分鐘)
- **目標**：使用 `input()` 收集本月收入與三項支出。
- **步驟**：
  1. 使用 `int(input("請輸入本月收入："))`
  2. 依序詢問 `房租、飲食、娛樂`
  3. 使用具描述性的提示文字
  4. 累計 `total_expense = rent + food + fun`
- **檢核點**：
  - [ ] 所有輸入轉成整數
  - [ ] 清楚說明金額單位（例如「元」）
  - [ ] 變數命名清楚 `income`, `rent_expense`, ...
- **範例輸出**：
```
請輸入本月收入 (元)：45000
請輸入房租支出 (元)：15000
請輸入飲食支出 (元)：9000
請輸入娛樂支出 (元)：3000
```

### 階段 C：判斷與鼓勵 (45 分鐘)
- **目標**：結合函式結果與條件判斷輸出鼓勵訊息。
- **步驟**：
  1. 呼叫 `calculate_savings`
  2. 以 `if/elif/else` 判斷儲蓄 > 0、== 0、< 0
  3. 印出對應訊息
  4. 顯示總結（收入、支出、儲蓄）
- **檢核點**：
  - [ ] 正儲蓄：印出「太棒了！繼續保持！」
  - [ ] 零儲蓄：印出「收支平衡，加油！」
  - [ ] 負儲蓄：印出「要小心，支出超標囉！」
- **範例輸出**：
```
=== 本月收支報告 ===
收入：45000 元
總支出：27000 元
儲蓄：18000 元
太棒了！繼續保持！
```

### 階段 D：進階挑戰 (可選 30 分鐘)
- **輸入驗證**：使用 `try-except`，避免非數字或負值。
- **摘要報表**：用 `print("-" * 30)` 製作表格效果。
- **儲蓄建議**：依儲蓄範圍提供建議，如「可設定 20% 自動儲蓄」。

## ⚠️ 常見錯誤

1. **忘記轉型**：`input()` 回傳字串，未轉換成整數導致加總錯誤。
2. **函式混合輸出**：`calculate_savings` 內部印出訊息，無法在主程式重複使用。
3. **條件少寫**：只判斷 `> 0` 與 `< 0`，忽略剛好 0 的狀況。
4. **命名含糊**：使用 `a`, `b`, `c` 表示支出，降低可讀性。
